## 5. 报告生成建议

### 5.1 通用过滤逻辑

建议报告生成时采用以下数据筛选逻辑：

1. 读取 `一线支持` 和 `二线支持` 两张表。
2. 按报告类型确定日期范围。
3. 优先筛选 `纳入汇报 = 是` 的记录。
4. 按 `日期`、`类别`、`系统名称`、`事件级别`、`业务影响`、`结果` 聚合。
5. 对 `工作内容` 进行摘要提炼，生成报告正文。
6. 对 `结果 = 延续`、`业务影响 = 已影响/有风险`、`事件级别 = 高/中` 的记录重点展示。

### 5.2 周报

建议统计维度：

- 本周总工作数。
- 本周纳入汇报工作数。
- 本周按类别统计。
- 本周重点事件和风险。
- 本周已完成事项。
- 本周延续事项。
- 下周需要关注的问题。

建议数据来源：

- `一线支持`
- `二线支持`

建议过滤：

- `日期` 落在周报周期内。
- `纳入汇报 = 是`。

### 5.3 月报

建议统计维度：

- 月度服务总量。
- 按类别统计服务内容。
- 按系统统计服务内容。
- 事件、变更、迁移、优化等重点服务类型汇总。
- 已完成和延续事项。
- 风险和业务影响事项。
- 交付文档清单。

建议数据来源：

- `一线支持`
- `二线支持`
- `服务文档`

### 5.4 季度报

建议统计维度：

- 季度总体服务情况。
- 各类别服务趋势。
- 生产系统/测试系统服务分布。
- 重点事件、重点变更、重点迁移和重点优化。
- 风险闭环情况。
- 下季度改进建议。

建议数据来源：

- `一线支持`
- `二线支持`
- `是否纳入汇报`

### 5.5 年度报告

建议统计维度：

- 全年服务总量。
- 各月份服务趋势。
- 各类别服务占比。
- 重大事件和风险汇总。
- 关键系统支持情况。
- 服务成果和改进成效。
- 下一年度规划建议。

建议数据来源：

- `一线支持`
- `二线支持`
- `服务文档`
- `是否纳入汇报`
