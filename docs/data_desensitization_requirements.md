# 文件脱敏生成需求文档

## 1. 需求背景

用户需要对指定目录下的文件进行批量脱敏处理。系统应读取某个源目录中的全部文件，并重新创建一个输出目录，将源目录中的文件按相同或可配置的目录结构生成对应的脱敏文件。

该功能用于在不暴露敏感信息的前提下，保留文件内容结构，方便后续查看、共享、测试或归档。

## 2. 需求目标

实现一个文件脱敏生成能力：

- 用户指定一个源目录。
- 系统在源目录的同级目录下自动创建一个新的目标目录。
- 系统遍历源目录下的文件。
- 系统对文件内容执行脱敏处理。
- 系统将脱敏后的内容写入自动生成的目标目录。
- 目标目录中的文件应尽量保持源文件的相对路径、文件名和文件格式。

## 3. 功能范围

### 3.1 输入

用户需要提供以下信息：

- 源目录路径：需要被脱敏处理的原始文件目录。

系统根据源目录路径自动推导目标目录路径。目标目录不要求用户手动指定。

可选配置包括：

- 脱敏规则：定义哪些内容需要被替换、隐藏或泛化。
- 文件处理范围：定义需要处理哪些文件类型，是否包含子目录。
- 目标目录命名后缀：默认使用 `_desensitized`。

### 3.2 输出

系统在源目录同级位置生成一个新的目标目录，目录中包含脱敏后的文件。

目标目录生成规则：

```text
target_dir = source_dir 的父目录 / (source_dir 名称 + "_desensitized")
```

示例：

```text
源目录：/Users/peyton/data/original_files
目标目录：/Users/peyton/data/original_files_desensitized
```

输出要求：

- 不修改源目录中的任何文件。
- 自动创建目标目录；如果目标目录不存在，应创建目录。
- 默认不允许用户指定任意目标目录，避免误写到不相关路径。
- 如果目标目录已存在，应根据配置决定覆盖、跳过、报错或追加时间戳。
- 脱敏文件应保留原始文件的相对目录结构。
- 脱敏文件应保留原始文件名，除非后续规则要求文件名也需要脱敏。

### 3.3 文件遍历

系统应支持：

- 处理源目录根目录下的文件。
- 递归处理子目录中的文件。
- 跳过隐藏目录、临时文件、缓存目录等可配置排除项。
- 遇到无法读取的文件时记录错误，不应导致整个任务直接中断，除非配置为严格模式。

### 3.4 文件类型

源目录下主要文件为 Office 文档，第一版应优先支持 Word、PPT、Excel 文件脱敏，同时兼容其他常见文本类文件。

核心支持文件：

- Word：`.docx`
- PowerPoint：`.pptx`
- Excel：`.xlsx`

Office 文件处理要求：

- Word 文件应脱敏正文、表格、页眉、页脚中的文本内容。
- PPT 文件应脱敏幻灯片文本框、表格、备注页中的文本内容。
- Excel 文件应脱敏单元格文本、公式中可能出现的敏感字面量、工作表名称中的敏感信息。
- 处理后应尽量保留原文件格式、排版、样式、图片、表格结构和工作表结构。
- 如果部分内容无法解析或无法写回，应记录在报告中。

扩展支持文件：

- PDF：`.pdf`
- 文本：`.txt`、`.log`、`.md`
- 数据文件：`.csv`、`.json`、`.xml`、`.yaml`、`.yml`
- 代码和脚本：`.py`、`.java`、`.js`、`.ts`、`.html`、`.sql`

PDF 文件处理策略：

- 第一阶段可先支持文本提取后生成脱敏文本副本，或在报告中标记为待处理。
- 如果要求输出仍为 PDF，需要单独实现 PDF 内容定位、覆盖和重写，复杂度高于 Word、PPT、Excel。

其他二进制文件处理策略：

- 未知二进制文件默认跳过，不复制到目标目录。
- 跳过原因应记录到处理报告中。
- 不建议原样复制未知二进制文件，避免将敏感内容带入脱敏目录。

默认建议：第一版优先实现 `.docx`、`.pptx`、`.xlsx`、`.txt`、`.csv`、`.json`、`.log`，其他格式作为扩展能力逐步支持。

## 4. 脱敏规则

### 4.1 基础脱敏对象

系统应支持对以下常见敏感信息进行脱敏：

- 手机号
- 身份证号
- 邮箱地址
- 银行卡号
- 姓名
- 地址
- IP 地址
- Token / API Key / Secret
- 密码字段
- 账号字段
- 数据库连接串
- URL 中的敏感参数

### 4.2 脱敏方式

可支持以下脱敏策略：

- 固定占位符替换，例如 `<PHONE>`、`<EMAIL>`、`<ID_CARD>`。
- 部分保留，例如手机号保留前三后四：`138****1234`。
- 哈希化，例如将原始值转换为不可逆摘要。
- 统一泛化，例如所有姓名统一替换为 `<NAME>`。
- 按字段名脱敏，例如 JSON 中 `password`、`secret`、`token` 字段统一替换为 `<REDACTED>`。

默认建议：优先使用固定占位符替换，便于确认脱敏覆盖范围，且不泄露原始数据。

### 4.3 示例规则

| 敏感类型 | 示例原文 | 脱敏结果 |
| --- | --- | --- |
| 手机号 | `13812345678` | `<PHONE>` |
| 邮箱 | `user@example.com` | `<EMAIL>` |
| 身份证号 | `110101199001011234` | `<ID_CARD>` |
| IP 地址 | `192.168.1.10` | `<IP>` |
| 密码字段 | `"password": "abc123"` | `"password": "<REDACTED>"` |
| Token 字段 | `token=abcdef123456` | `token=<REDACTED>` |

## 5. 处理流程

1. 校验源目录是否存在且可读取。
2. 根据源目录自动计算同级目标目录路径。
3. 校验目标目录是否可创建或可写入。
4. 加载脱敏规则配置。
5. 遍历源目录下的文件。
6. 判断文件是否属于可处理范围。
7. 对文本文件读取内容。
8. 按脱敏规则替换敏感信息。
9. 在目标目录中创建对应的相对路径。
10. 写入脱敏后的文件。
11. 生成处理报告。

## 6. 处理报告

建议每次执行后生成处理报告，便于确认结果。

报告内容包括：

- 源目录路径。
- 目标目录路径。
- 开始时间和结束时间。
- 总文件数。
- 已处理文件数。
- 已跳过文件数。
- 失败文件数。
- 每类脱敏规则命中次数。
- 跳过文件列表。
- 失败文件列表及失败原因。

报告可保存为：

- `desensitization_report.json`
- 或 `desensitization_report.md`

## 7. 异常处理

系统应处理以下异常情况：

- 源目录不存在。
- 源路径不是目录。
- 目标目录已存在。
- 文件无读取权限。
- 文件编码无法识别。
- Office 文档解析失败。
- Office 文档写回失败。
- 文件过大。
- 文件内容为空。
- 脱敏规则配置错误。
- 输出文件写入失败。

默认处理建议：

- 源目录异常：直接失败并提示。
- 目标目录已存在：默认报错，避免覆盖已有结果。
- 单个文件失败：记录失败并继续处理其他文件。
- 编码无法识别：跳过并记录。
- Office 文档解析或写回失败：记录失败原因，并继续处理其他文件。
- 未知二进制文件：跳过并记录。

## 8. 配置项

建议支持以下配置项：

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| `source_dir` | 源目录路径 | 必填 |
| `target_suffix` | 自动生成目标目录时追加到源目录名后的后缀 | `_desensitized` |
| `target_dir` | 目标目录路径 | 自动推导，默认不要求用户传入 |
| `recursive` | 是否递归处理子目录 | `true` |
| `overwrite` | 目标目录存在时是否覆盖 | `false` |
| `existing_target_strategy` | 目标目录已存在时的处理策略，可选 `error`、`skip`、`overwrite`、`timestamp` | `error` |
| `include_extensions` | 需要处理的文件扩展名 | `.docx`、`.pptx`、`.xlsx`、`.txt`、`.csv`、`.json`、`.log` |
| `exclude_patterns` | 排除文件或目录规则 | `.git`、缓存目录、临时文件 |
| `binary_strategy` | 未知二进制文件处理策略 | `skip` |
| `report_enabled` | 是否生成处理报告 | `true` |
| `strict_mode` | 遇到单文件错误是否中断任务 | `false` |

## 9. 非功能需求

### 9.1 安全性

- 不应修改源文件。
- 不应在日志中输出原始敏感值。
- 处理报告中只记录命中次数和文件路径，不记录原始敏感内容。
- 默认不复制无法识别类型的文件，避免敏感信息泄露。

### 9.2 性能

- 应支持批量处理目录中的多个文件。
- 对大文件应有大小限制或流式处理机制。
- 对异常文件应快速跳过，避免阻塞整个任务。

### 9.3 可维护性

- 脱敏规则应可配置。
- 文件类型白名单应可配置。
- 处理流程应便于扩展新的脱敏规则。

## 10. 验收标准

功能验收：

- 给定一个存在的源目录，系统可以在源目录同级位置创建新的目标目录。
- 新目标目录名称默认为源目录名称加 `_desensitized` 后缀。
- 目标目录中的文件数量和可处理文件数量一致。
- 目标目录中的文件相对路径与源目录保持一致。
- 源目录文件不会被修改。
- Word、PPT、Excel 文件中的手机号、邮箱、身份证号、密码、Token 等敏感信息被替换。
- 文本、CSV、JSON、日志等扩展文件中的敏感信息被替换。
- Office 文件脱敏后应尽量保留原始格式、排版、样式和结构。
- 无法处理的文件会被跳过并记录。
- 任务完成后生成处理报告。

安全验收：

- 脱敏输出文件中不应出现已定义规则覆盖的原始敏感信息。
- 日志和报告中不应出现原始敏感值。
- 目标目录已存在且未开启覆盖时，不应覆盖已有文件。

## 11. 待确认问题

以下问题需要在实现前确认：

1. 需要处理的源目录路径是否由命令行参数传入，还是在界面中选择？
2. 目标目录名称是否固定使用 `_desensitized` 后缀，还是允许配置后缀？
3. 是否需要保留源目录完整结构？
4. 文件名本身是否也需要脱敏？
5. 未知二进制文件应该跳过、复制，还是记录后忽略？
6. PDF 是否需要输出为脱敏后的 PDF 原格式，还是允许转换为脱敏文本文件？
7. 姓名、地址等非结构化信息是否需要识别？如果需要，是否接受误判风险？
8. 脱敏规则是否需要可逆？如果需要，应改为加密或映射表机制。
9. 是否需要生成脱敏前后对比报告？
10. 是否需要命令行工具、桌面工具，还是集成到现有项目中？

## 12. 初始实现建议

建议第一版实现为命令行工具，优先支持 Word、PPT、Excel 文件，同时支持常见文本类文件。

推荐能力：

- 支持必填参数：`source_dir`。
- 自动生成目标目录：源目录同级路径下的 `{源目录名}_desensitized`。
- 可选支持参数：`target_suffix`、`existing_target_strategy`。
- 递归扫描源目录。
- 优先处理 `.docx`、`.pptx`、`.xlsx` 文件。
- 同时处理 `.txt`、`.csv`、`.json`、`.log` 等常见文本类文件。
- 使用正则规则脱敏手机号、邮箱、身份证号、IP、密码、Token、Secret。
- 跳过未知二进制文件，避免未脱敏内容进入输出目录。
- 生成 JSON 格式处理报告。
- 默认禁止覆盖已有目标目录。

PDF、老版本 Office 文件（`.doc`、`.ppt`、`.xls`）和其他复杂二进制文件建议放到第二阶段处理。第一版优先处理新版 Office Open XML 文件（`.docx`、`.pptx`、`.xlsx`），因为它们更适合在保留格式的前提下进行内容替换。
